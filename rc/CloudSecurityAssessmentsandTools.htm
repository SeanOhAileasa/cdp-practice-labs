Welcome to the Cloud Security Assessments and Tools Practice Lab. In this module, you will be provided with the instructions and devices needed to develop your hands-on skills.

As a cloud security specialist, a crucial part of your skillset would be the ability to detect software vulnerabilities in an infrastructure. As technology advances, there has been a shift in the way in which cyberattacks are conducted.

With companies moving more and more resources to cloud service providers; or making use of hosted infrastructures for specific web applications, it has become crucial that these applications are secure to prevent exploits from cyber security criminals.

In cloud environments, you may choose to perform a Network-based scan, which allows you to easily scan an entire network from a singular host to discover potential risks, or potentially run an Agent-based scan, which instead can be broken up and run on many different hosts, provided that an agent has been installed.

Another method of ensuring that updates and patching completes as smoothly as possible is by implementing Scheduled updates. When enabled, scheduling different updates will stop devices updating automatically (or not at all) and allow administrators to better manage the risk imposed by a particular update.

In this lab, the Damn Vulnerable Web Application (DVWA) will be used to demonstrate how vulnerabilities can be detected by using a software vulnerability scanner. This can also be used in public cloud infrastructures such as Azure.

Task 1 - Verify that DVWA is running using AlienVault

The Damn Vulnerable Web Application (DVWA) is a free, open-source application that can be used by security professionals to practice and enhance their skillset in a secure legal infrastructure.

In this task, the DVWA application will be started to ensure it will be accessible to conduct a software vulnerability scan.

Step 1

Connect to PLABWIN10.

Step 2

Click Start and select XAMPP in the Start menu.

Note: The XAMPP application is used to host an Apache web application on Windows and Linux operating systems. This application also configures the necessary MySQL database for the installation. In the infrastructure, the application has already been configured to run the DVWA.

Step 3

In the XAMPP Control Panel in the Actions column, select Start for the following services:

Apache

MySQL

Note: The Apache and MySQL services need to run for the DVWA to be accessible.

Step 4

Minimize the XAMPP Control Panel.

Note: After starting the specific services, it can be seen which specific ports are open for the DVWA to function. Port 80 and 443 are used for web application traffic on the network, where port 80 uses the http protocol, and port 443 uses the HTTPS protocol. The HTTPS protocol is more secure as it encrypts the network, whereas the HTTP protocol does not encrypt the traffic.

Step 5

From the Taskbar, open Microsoft Edge.

Step 6

Using Microsoft Edge, Navigate to the following website:

http://192.168.0.4/dvwa

Step 7

Use the following credentials to log into the web application and click Login:

Username: admin

Password: password

Step 8

In the Microsoft Edge pop-up window, select Never.

Step 9

In the DVWA Web Application, select DVWA Security in the left-hand side pane.

Step 10

On the DVWA Security page, change the security level to Low and click Submit.

Note: The DVWA web application has several security levels. These levels can be changed depending on the type of security exercise that needs to be conducted. Leave the devices in their current states and continue on to the next task.

Task 2 - Conduct a Software Vulnerability Scan Using AlienVault

In this task, a software vulnerability scan will be conducted on DVWA (Damn Vulnerable Web Application) using AlienVault.

AlienVault is a free, open-source Security Information and Event Management (SIEM) tool. AlienVault can be used to detect software vulnerabilities in web applications; these detected vulnerabilities need to be rectified by a cloud security professional to ensure the environment is secure from exploits and potential data breaches.

AlienVault will be accessed through a local browser remotely, accessing the managing interface.

Step 1

Power on PLABDM01 and PLABALIENVAULT.

Alert: Please allow 2 minutes to pass before continuing. This ensures enough time has been provided for PLABALIENVAULT to complete its boot cycle. This is important as Step 3 uses the address: 192.168.0.6, which is for PLABALIENVAULT.

Connect to PLABDM01.

Step 2

In the Taskbar, open Firefox.

Step 3

Navigate to the following URL:

https://192.168.0.6

Step 4

In the Warning: Potential Security Risk Ahead tab of Firefox, select Advanced…

Note: An error message displays that the connection is not secure. This message can be safely ignored as the website uses a self-signed web certificate. In a production environment, a third-party certificate will be installed to ensure the website can be accessed securely.

Step 5

In Firefox, select Accept the Risk and Continue.

Step 6

Enter the following credentials and click Login.

USERNAME: admin

PASSWORD: Passw0rd

Step 7

In the Dashboard of the AlienVault website, hover on Environment and select Vulnerabilities.

Note: The AlienVault SIEM application can be used for several different types of vulnerability scans; for this particular exercise, a software vulnerability scan will be conducted.

Step 8

On the Vulnerabilities window, select Scan Jobs.

Step 9

In the Scan Jobs window, select New Scan Job.

Step 10

In the Create Scan Job pop-up window, add the following to the Job Name field.

PLAB Vulnerable Scan

Step 11

In the Type here to search assets field, enter the following and press Enter:

192.168.0.4

Step 12

Select Save in the Create Scan Job window.

Note: The specific IP address that was entered is PLABWIN10’s IP address, which hosts the DVWA application.

Step 13

Select the newly created PLAB Vulnerable Scan. This will highlight the scan job in blue.

Note: You may need to select a different tab, such as Threat Database, and then click back onto the Scan Jobs tab for the scan to start.

Note: The scan job may take up to 15 minutes to complete.

Step 14

After the scan has been completed, select the View report, which is shown as a PDF icon in the Action column.

Note: There are several options available in the ACTION column that you can explore for a better understanding of how to use Alien Vault.

Step 15

Your report will appear in a separate tab.

After the vulnerability scan has been conducted, a report is automatically generated to display all the vulnerabilities that were detected on the Damn Vulnerable Web Application (DVWA).

This specific report can be used by cloud security professionals to rectify the newly discovered vulnerabilities. In the report suggested rectification of the vulnerabilities is also generated.

Note: Keep the XAMPP Control Panel open, as you’ll be using this in the next exercise.

Task 1 - Conduct a Software Vulnerability Scan using Nessus

Nessus is a vulnerability scanner that can be used to scan web applications for software vulnerabilities. A free trial version of the application is available to cloud security professionals, which can be used to test the functionality of a software application.

In this task, the Damn Vulnerable Web Application will be scanned for software vulnerabilities using the Nessus vulnerability scanner.

Step 1

Connect to PLABWIN10.

Note: The XAMPP Control panel is still open from the previous exercise. If the services are not running or the exercise was started before Exercise 1 was completed, please repeat Task 1 from Exercise 1 before carrying on with this exercise.

Step 2

Make sure you have your Microsoft Edge browser open.

Reopen Microsoft Edge by clicking the icon on the Taskbar if required.

Step 3

Navigate to the following URL in Microsoft Edge:

https://localhost:8834/

Note: By browsing to the specific URL, the web interface for the Nessus vulnerability scanning application will be opened.

Step 4

In the Privacy error window, select Advanced.

Note: An error message displays that the connection is not secure. This message can be safely ignored as the website uses a self-signed web certificate. In a production environment, a third-party certificate will be installed to ensure the website can be accessed securely.

Step 5

In the Privacy error window, select Continue to localhost (unsafe).

Step 6

Enter the following credentials and select Sign In.

Username: PLABAdmin

Password: Passw0rd

Step 7

In the Nessus Web interface, scroll to the right and select New Scan.

Step 8

In the Scanner window, select Web Application Tests.

Step 9

In the New Scan/Web Applications window, complete the following fields and click Save.

Name: PLAB Vulnerable Scan

Targets: 192.168.0.4

Note: The specific IP address that was added in the Targets field is that of the PLABWIN10 machine, which hosts the DVWA application.

Step 10

In the My Scans window, select the newly created PLAB Vulnerable Scan.

My Scans can be found from the FOLDERS menu on the left-hand side.

Step 11

Scroll to the right of the webpage and select Launch.

Note: Please allow Nessus time to complete the scan; this typically takes around 5 minutes.

Step 12

Once the scan has been completed, scroll to the right, click the Report menu and select HTML.

Note: Your results may be different from what is shown in the screenshots.

Step 13

In the Generate HTML Report pop-up window, select Generate Report.

Step 14

The report will start to download and appear at the bottom of the screen.

Once completed, click Open file and select the option to open in Microsoft Edge.

Step 15

As you can see in the report, Nessus has detected multiple vulnerabilities, the majority of these being set under the Info tab. Typically, you would, of course, hope to see little (potentially a few Low or Medium risk vulnerabilities) or no errors in the report.

If there are preventable vulnerabilities detected, you can then use this to escalate to the correct channel inside of an organization, and review these in more detail for potential security fixes.

Select Show Details in the newly opened Microsoft Edge tab.
