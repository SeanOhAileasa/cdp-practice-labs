Exercise 1 - Provisioning a S3 Bucket

Amazon Simple Storage Service, better known as Amazon S3 is a service offered within AWS.

S3 provides high durability, high availability, and high performance, making it a massively scalable storage solution.

In this exercise, you will provision a S3 bucket within AWS.

Learning Outcomes

After completing this exercise, you should be able to:

  • Create an AWS S3 Bucket

Your Devices

You will be using the following device in this lab. Please power this on now.

4153b532-d93b-4759-b999-5667d2abe624

Task 1 - Create an S3 Bucket

Objects are entities stored in Amazon S3, such as files and the associated metadata that describes those files.

Once an object is uploaded to S3, it must be stored in a container known as an S3 bucket.

In this task, you will create an S3 bucket an upload an object.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, click Microsoft Edge.

Step 3

From the AWS login page, click Sign In

Step 4

Click on Services in the top left corner, and scroll down the list and select Storage.

Step 5

From the Storage submenu, select S3.

Step 6

From the Amazon S3 > Buckets page click Create bucket.

Step 7

In the Bucket name textbox type:

plabbucket00

Ensure AWS Region is set for EU (Ireland) eu-west-1.

Note: If a message appears stating the bucket name already exists, increase the last number e.g. change to 01.

Note: The Bucket name you create can’t be made up of spaces or uppercase letters and needs to be unique.

Step 8

Scroll down to the end of the page and click Create bucket.

Step 9

It will return to Amazon S3 > Bucket.

Notice the newly created plabbucket00 is on the list.

Step 10

Click plabbucket00.

Step 11

From the Objects tab, click Upload.

Step 12

From the Upload page, click Add folder.

Step 13

From the Open dialog box, go to This PC > Local (C:) > Program files.

Select Windows Media Player.

Click Upload.

Step 14

On the pop-up message, click Upload.

Step 15

Click Upload.

Step 16

A message will appear stating the Upload succeeded.

Click Close.

Step 17

Notice on the Objects tab, the Windows Media Player folder is now listed.

Click the Properties tab.

  • Bucket versioning

Exercise 2 - Provisioning Azure Storage Account Resources

Azure offers various solutions for provisioning storage to meet any environment’s needs. Each solution optimized for its intended function, grants system administrators access to massively scalable, highly available, durable, and secure storage.

In this exercise, you will provision a storage account and configure a blob container and Azure fileshare.

Learning Outcomes

After completing this exercise, you should be able to:

  • Provision a Storage Account in Azure
  • Create a blob container
  • Create an Azure Fileshare

Your Devices

You will be using the following devices in this lab. Please power this on now.

af4fa57a-53be-4f51-bf97-9ac3495df23

Task 1 - Provision a Storage Account in Azure

Creating the required storage necessary for any environment in Azure is a quick process.

As the provisioning of these resources are highly modularized and flexible, they can be created pre-emptively, or on the fly.

In this task, you will create a storage account.

Step 1

Connect to PLABWIN10.

Note: When opening the Microsoft Edge browser, it will redirect you to the Azure portal and sign in automatically.
An Azure subscription will be provided for all lab activities that utilize the Azure platform.
Note that in a real-world scenario, either a subscription will need to be purchased or will already exist as part of an existing company infrastructure.

Step 2

From the Taskbar, click Microsoft Edge.

Step 3

In the Azure Portal, click into the Search resources, services and docs textbox and type:

storage

Under the results for Services, select Storage accounts.

Step 4

From the Storage accounts page, click Create.

Step 5

From the Create a storage account page, scroll down to the Instance details section, click into the Storage account name textbox and type:

cloudplusstorage01
Note: Names should be 3-24 characters, and can only contain lowercase letters and numbers. If a message states it already exists, increase the number e.g to 02.

Step 6

From the Region dropdown, select (Europe) UK South.

Step 7

Under Performance, ensure the Standard option is selected.

From the Redundancy dropdown, select the Locally-redundant storage (LRS).

Click Next : Advanced >.

Step 8

In the Advanced tab, ensure the checkboxes for REST API, blob public access and key access are ticked.

Keep the default options for the Minimum TLS version.

Click Next : Networking >.

Step 9

From the Networking tab, under Network connectivity > Network access, ensure Enable public access from all networks is selected.

Step 10

Under the Network routing > Routing preference, ensure Microsoft network routing is selected.

Click Next : Data protection >.

Step 11

From the Data protection tab, click Review.

Alert: ensure that all checkboxes on this page have not been checked.

Step 12

From the Review tab, click Create.

Step 13

Notice the Deployment is in progress message is displayed.

Note: Depending on the options you have set, in a real-life scenario, it could take a while to provision the storage account.

Step 14

Notice that the message Your deployment is complete has appeared.

Keep PLABWIN10 in its current state for the next task.

Task 2 - Create a Blob Container

Certain forms of data don’t always fit a specific mold or scheme, for files that tend to be more diverse, provisioning a blob container is highly recommended.

While highly flexible, blob containers can also be attached to Virtual machines directly as mapped storage, or accessed by utilizing tools like Azure Storage Explorer

In this task, you will provision a blob container.

Step 1

Ensure you are connected to PLABWIN10 with the Azure portal window open to the cloudplusstorage01 page.

Click Go to resource.

Step 3

Under the Data storage menu, click Containers.

Step 4

Click Container.

Step 5

In the Name textbox, type:

cloudpluscontainer

Keep Public access level as Private.

Click Create.

Step 6

Now that we have a container, we can look to upload information into the cloud.

Click on the newly created cloudpluscontainer container.

Step 7

Within cloudpluscontainer, click Upload.

Step 8

Under Upload blob, click on the blue folder icon to browse for a file.

Step 9

Go to This PC >Downloads and select example.txt.

Click Open.

Step 10

Once selected, click Upload.

Step 11

Close the Upload blob window using the X icon.

Once uploaded, notice that the metadata is also visible. You can see file information such as Modified date, Access tier and Blob type.

Click the example.txt file.

Step 12

Notice that full metadata details for this file is shown.

Note: The Change tier option is also available from this page. This can be helpful for example, to amend files that should now be moved into Archive. Keep PLABWIN10 in its current state for the next task.

Task 3 - Create an Azure Fileshare

Covered in a previous module was the creation of a file share on an on-premises environment.

Azure offers a cloud solution that allows more scalability when setting up a file share, whether the intention is to replace or supplement an existing on-premises file servers.

Offering fully managed file shares accessible through multiple industry standards; Azure Files Rest API, NFS, and SMB protocols.

In this task, you will provision a fileshare.

Step 1

Reconnect to PLABWIN10.

The Azure portal window will be open on the cloudplusstorage01 > example.txt page.

Click the cloudplusstorage01 breadcrumb weblink.

From the cloudplusstorage01 | Containers page, under the Data storage menu, click File shares.

Step 3

Click File share.

Step 4

In the Name textbox, type:

cloudplusfileshare

Keep Tier as Transaction Optimized.

Click Create.

Step 5

Click the three dots on the fileshare cloudplusfileshare, and select Edit quota.

Step 6

In the Quota textbox replace the existing value with:

2000

Click OK.
