Exercise 1 - Exploring Integration in Cloud Environments

When it comes to integrating a cloud environment, an organization needs to consider what kind of functions the infrastructure will serve, and what kinds of resources are needed.

This could include things such as Software as a Service (SaaS) subscriptions for office-based software, storage media and e-mail messaging, as well as collaboration tools and directory services.

In this exercise, we will explore a common SaaS subscription service used by organizations, Microsoft 365.

Task 1 - Explore Microsoft 365

External to tools provided by public cloud infrastructure, there are also several different services that can be employed for private or non-cloud-based infrastructures.

One of these is Microsoft 365, which provides different tools to businesses through a subscription model.

In this task, we will explore Microsoft 365 and the SaaS solution it provides businesses.

Note: A Microsoft 365 license is needed to further explore the topics covered in this exercise. In a real-world scenario, it is likely that a business would purchase a license key to each piece of software and install them as needed.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, select Microsoft Edge.

Step 3

In the Intranet webpage, in the Address bar, type the following:

microsoft 365

Press Enter.

Step 4

Scroll down the list of results and select the Microsoft 365 - Subscription of Office Apps | Microsoft 365 weblink.

Step 5

This will bring you to the Microsoft 365 homepage.

From the Cookie notice at the top of the page, click Accept.

Note: This window may not appear, depending on the IP address the lab environment uses. The URL may also default to the UK version of the webpage, in which case the en-GB section will be replaced with en-us. These websites are practically identical, although the currencies will change depending on your location.

Step 6

Go to the Find the right plan for you section.

Click Business.

Step 7

Scroll down to view the For business tab.

Here, you will see the different service tiers and subscription offerings by Microsoft.

Note: Microsoft 365 is classified as SaaS due to the subscription-style arrangement to be able to use the software. Microsoft 365 is possibly the most common SaaS used across the world, as many different businesses require the use of general Office software such as Outlook and Excel to function.

Step 8

Notice they show the different software included in each bundle, ramping up with each tier.

These are as follows (at the time of this module development), from each tier:

  • Word: a globally accepted document processor.
  • Excel: a globally accepted spreadsheet processor.
  • PowerPoint: a globally accepted PowerPoint creator.
  • Teams: a work collaboration tool, allowing Instant Messaging and allowing conference calls and meetings.
  • Outlook: a globally accepted E-mail program, which works in tandem with Exchange.
  • SharePoint: a SaaS storage location to allow for easy document sharing and collaboration.
  • Exchange: a cloud-based e-mail infrastructure tool, typically used in private environments.
  • OneDrive: another SaaS storage location.
  • Access: a program that allows the creation of databases.
  • Publisher: a program that allows more creative document design, intended to be used for things such as marketing materials.
  • Intune: a cloud-based service that allows Mobile Device Management (MDM) and Mobile Application Management (MAM)
  • Azure Information Protection: a cloud-based service that is used to classify, identify and protect sensitive data.
Note: Once a subscription is purchased, the most recent and up-to-date versions of these tools are provided. You can see which products are offered with each tier using this tool. In a real-world scenario, you would decide which is the best tool for your business before selecting a tier. It could be that Microsoft Intune and Azure information protection are not necessary, so you may go with the Business Standard tier.

Step 9

Under the Explore Microsoft 365 heading, click the Learn more about Microsoft 365 for business heading > Learn more web link.

Step 10

Review the information on why you may wish to implement Microsoft 365 into your infrastructure and wider business.
