Exercise 1 - Using Active Directory Administrative Center

Management of dynamic environments presents various challenges, as an organization’s network complexity tends to grow over time, adding requirements to address network security and maintaining a coherent record of IT infrastructure.

Active Directory Administrative Center can be used to enforce a logical approach to manage network object access and store necessary information about these objects as they relate to an organization’s hierarchical structure. Utilizing this native Active Directory tool in Window Server Management proves a means for administrator and security engineers to maintain logical user access, while following Least Privilege Access.

In this exercise, the features and functionality of Active Directory Administrative Center will be explored.

Learning Outcomes

After completing this exercise, you should be able to:

  • Create Active Directory Groups and add Members to these new groups
  • Create a File Share and configure access permissions

Task 1 - Create and Populate a New Active Directory Group

Using Active Directory Administrative Center, administrators and security engineers have access to the following:

  • Control over user access through logical hierarchical groups.
  • Management of Role-based Access Controls for groups and users, to implement mandatory access control and discretionary access control.

They can also:

  • Allocate and deallocate resource access to users based on security groups following Least Privilege Access.
  • Programmatically configure the distribution of Group Policy.

In this task, Active Directory Administrative Center will be used to create a Security Group and populate a user within it.

Task 2 - Creating and Assigning Permissions to New File Share

Using File Share Services in Server Manager, administrators and security engineers have access to the following:

  • Control over user access to shared folder resources.
  • Services to create logical folder IT infrastructure to shared resources

They can also:

  • Allocate and deallocate resource access to users based on security groups following Least Privilege Access.
  • Grant network access to shared resources stored on a remote server.

In this task, File Share Services will be used to create a file share and populate it with an active directory group created in a previous task.

Step 5

The New Share Wizard will give you five options when creating a new share, Server Message Block (SMB) Quick, Advanced and Share Application, and Network File System(NFS) Quick and Advanced. The Differences are as follows:

  • SMB Share Quick: a standard file sharing used by all versions of Windows. SMB Share Quick enables management of full share and NTFS permissions.
  • SMB Share Advanced: enables SMB sharing with full share and NTFS permission as well as services provided by File Server Resource Manager.
  • SMB Share Application: Provides SMB sharing with settings aligned for use with Hyper-V and other applications.
  • NFS Share Quick: a standard file sharing protocol used extensively by UNIX and Linux systems. NFS Share Quick enables NFS sharing with authentication permissions and access to services provided by File Resource Manager.
  • NFS Share Advanced: provides NFS Sharing with authentication permissions and access to services provided by File Resource Manager.

Ensure that SMB Share - Quick option is highlighted.

Exercise 2 - Using Azure Key Vault

Hosting and managing certain resources in Azure (for example, a web application) would add a requirement to securely store credentials to allow access.

Azure Key Vault can be used to store these security credentials without the need for a third-party key management suite. Utilizing the native suite of tools available in Azure will ensure the ease of manageability and compatibility of resources.

In this exercise, the features and functionality of Azure Key Vault will be explored.

Learning Outcomes

After completing this exercise, you will be able to:

  • Enable and Configure Azure Key Vault
  • Create Security Keys, Secrets and Certificates using Azure Key Vault

Task 1 - Create and Populate Azure Vault

Using Azure Key Vault, administrators and Security Engineers have access to the following:

  • Control over all access keys and passwords
  • Control over Hardware Security Modules (HSMs), including the ability to maintain and configure them as necessary

They can also:

  • Ensure that applications used on Azure have no access to keys
  • Automate tasks for SSL and TLS Certificates

In this task, Azure Key Vault will be used to create a vault and populate several items within it.

Step 14

Azure will give you two encryption options when creating keys, Rivest, Shamir and Adleman (RSA for short, named after the creators of the method), and Elliptic Curve (EC). The differences are as follows:

RSA keys refer to a type of encryption where the RSA algorithm is used to generate a pair of public and private keys. This is the typical encryption method used by SSL certificates for websites.

EC Keys refer to a type of encryption where the same pair of keys are generated via an elliptic curve using an algebraic formula.

For an example, let’s use an RSA key. Azure also asks what size you would like this key to be, which notes how many bits are used in the encryption method. Keep the default size of 2048.

When selecting a certificate authority, Azure provides you with three options:

Self-signed certificates are signed by the creator of the signature, and as such, have not been verified by a certification authority. A browser that detects a self-signed certificate may warn users that the certificate cannot be verified before accessing the website.

Certificates issued by integrated CA’s are certificates that have been authorized by CA’s that Key Vault uses, for example, DigiCert and GlobalSign.

Certificates issued by a non-integrated CA refers to any certificate certified by a third-party CA external to Azure.
