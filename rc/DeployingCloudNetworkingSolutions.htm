Exercise 1 - Exploring Virtual Private Cloud

Virtual private clouds are computer resources and services offered over the Internet or on a private internal network, not accessible by the general public.

Like the public cloud, businesses gain the benefits of self-service, scalability and elasticity while retaining control and customization from these dedicated resources.

Infrastructure as a Service (IaaS) and Platform as a Service (PaaS) are the two models offered as cloud services in a private cloud.

With IaaS, a cloud computing service provider manages the infrastructure, Leaving the tenant to manage their software's license, installation, configuration, and management.

Task 1 - Configure A Hub and Spoke Network

A Hub and Spoke network (also known as a star network) refers to the logical layout of a network. This Site-to-site topology has a central device called the hub and connects to various devices referred to as spokes.

Large organizations often have different offices in very different geographical locations, anywhere from within the same city to global locations.

This topology is normally used to keep all the spokes sites connected to a central hub. With Azure, a virtual hub still acts as the central point of connectivity for all spoke virtual networks, with the spokes peering with the hub to isolate workloads.

Task 2 - Deploy a Virtual Network Gateway

A virtual network is not very different from an on-perm network that utilizes switches and routers. These virtual network gateways are created using two or more VMs by Azure.

Automatically configured and deployed, you don’t have direct access to these VMs, as they are part of Azure's internal infrastructure. The settings you select when configuring your gateway will impact the configuration and deployment of these VMs.

Task 3 - Configure a Load Balancer

Virtual Machines and Networks are efficient at processing workloads and job requests to the extent of their physical capacities.

As network traffic and general application usage grows, that limit will bottleneck workloads, slowing response times. The solution to this situation would be evenly distributing the workloads across various resources.

One convenient aspect of load balancers is that workloads can be distributed to resources based on their needs. Such as directing all HTTP requests to a set of VMs that will handle website traffic that might require extensive resources while directing SQL traffic to a dedicated VM that might not have as heavy workloads.
