Exercise 1 - Securing Cloud Infrastructure Resources

Regardless of an organization’s preferred environment, on-premises or on cloud, one aspect remains consistent, the need to secure resources. Cloud security spans over several tools, protocols and best practices; familiarizing yourselves with these will enable your ability to address these needs.

Securing cloud infrastructure resources does not end at the initial configuration. Any change brings risk. Even if the environment is secured today, it may not be tomorrow.

In this exercise, you will explore various tools used to secure infrastructure resources.

Task 1 - Configure a Group Policy

Group policy is a Windows Administration tool that allows System Administrators and Security Engineers to configure various settings across an organization’s infrastructure.

While exploring some of the features of this administrative tool, it should be noted how extensive and thorough some of the policies can extend to.

Any changes to group policy should be first tested in a controlled environment before pushing updates to production.

Given the extensive nature of this administrative tool, this task will cover some more commonly used group policies.

In this task, you will configure group policies using the Windows Administration tool.

Task 2 - Explore Event Viewer

Event viewer is a native tool to windows, used to log application and system operational data from errors to informational messages.

This tool is used for various forms of troubleshooting Windows systems and applications that run on them.

In this task, you will get a quick overview of where this tool is located and the kind of information it displays.

Step 1

Ensure that you are connected to PLABDC01.

Step 2

From Server Manager > Dashboard, click Tools and select Event Viewer

Step 3

From the Event Viewer window, expand Windows Logs and select System

Step 4

Right-click System and select Filter Current Log

Step 5

Under Event Level, tick the checkboxes for Critical, Warning and Error.

Click OK.

Step 6

Notice that the log entries will now filter to only display Error, Warning and Critical event level type messages.

Click any of the error messages.

Under Event 144, Time-Service notice in the General tab, it gives you a brief description of the error being examined, as well as the time it was generated

Note: The Level column indicates what type of log it is. The most common type is Information. The Information log will notify you when something has been changed, and an app or service is logging an event. A Warning log notifies you that something might not be quite right. An Error log is notifying you that something unexpected happened, whereas a Critical log will notify you when something is down, broken or crashed and needs urgent attention.

Step 7

From the expanded Windows Logs, click Security, then right-click Security and select Filter Current Log

Step 8

From the Filter Current Log dialog box, expand the Keywords dropdown and tick the Audit Failure checkbox.

Then, click OK.

Step 9

The log will generate any failed instances of login attempts

Note: There’s a possibility that no logs have been generated for this section yet. But it’s important to take note that these security logs are useful in determining failed login attempts by malactors or even users who have forgotten their passwords or attempting to access resources they do not have permissions for. 7d0ac9fb-5de8-4057-8699-2545ccd83fb6

Task 1 - Encrypt an Azure VM OS and Data Disks

As more organizations move to the cloud, it’s easy to forget, resources in Azure still have a physical form. Housed in Microsoft datacenters, the newly spun VM is created on a physical device. Encrypting resources adds additional layers of security to these devices.

In this task, you will learn how to encrypt an Azure VM’s OS and data disk.
