Exercise 1 - Examining and Configuring Dashboard and Reports

698c76b2-606c-4d2a-af5a-9308bfd16e35

As you can see, utilizing dashboards and reporting is very important. For example, a Cloud Administrator will find themselves spending a lot of time working on their professional life.

Configuring dashboards and creating reports can be one of the most fun and rewarding aspects of this career path.

Reporting plays a vital role in showing us aspects of our environment that we don’t get from a dashboard, or what happens when we aren’t in front of a dashboard, out of hours.

For example, imagine you look at bandwidth usage metrics and see that at 4:00 am every day, the NIC on a server is saturated with traffic for a little over an hour, and then by carrying out a review, find out that’s when someone like a DB admin is running their offsite backups.

These are examples of how dashboards and reporting work together.

In this exercise, you will configure and deploy resources with tagging within Microsoft Azure. You will also utilize the Azure dashboard and filter resources by the tags you assign to them.

After completing this exercise, you should be able to:

Task 1 - Configure Resources to view in the Dashboard

In this task, you will configure and deploy resources using tagging within Azure. You will also utilize the Azure dashboard and filter resources by the tags you added to them.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, click Microsoft Edge.

Note: When opening the Microsoft Edge browser, it will automatically redirect to the Azure Portal and sign in. An Azure subscription will be provided for all lab activities.

Step 3

From the Azure portal page, click Create a resource.

Step 4

From the Create a resource page, under the Storage account resource option, click the Create weblink.

Step 5

The Create a storage account page will appear.

Ensure the Resource group dropdown is showing the pre-populated resource group.

Step 6

From the Instance details section, click in the Storage account name textbox and type:

plstoreone

Step 7

Click in the Region dropdown box and select (Europe) UK South option.

Step 8

From the Performance, ensure that the Standard option is selected.

Step 9

Click the Redundancy dropdown and select Locally-redundant storage (LRS).

Step 10

Click the Review button.

Step 11

From the Validation page, click the Create button.

Step 12

The Overview page of the storage resource will load, and the Deployment is in progress information will be displayed.

Step 13

The Overview page will display the message Your deployment is complete when it’s finished.

Step 14

Click on the Home weblink.

Step 15

Click the Create a resource button.

Step 16

From the Create a resource page, under Virtual machine, click the Create weblink.

Step 17

The Create a virtual machine page will appear.

Click the Resource Group dropdown menu and select the pre-populated option.

Step 18

From the Instance details section, click into the Virtual machine name textbox and type:

practicevm01

Step 19

Ensure the Region dropdown shows (Europe) UK South as selected.

Step 20

Still under Instance details > Image, click the See all images weblink.

Step 21

The Select an image webpage will be displayed.

Under Microsoft Windows 10, click the Select dropdown.

Step 22

From the Select dropdown list, find the Windows 10 Pro, version 21H2 - x64 Gen 1 image.

Step 23

You will now be back at the Create a virtual machine webpage. Scroll down to the Administrator account section.

Under Size, click the See all sizes weblink.

Step 24

From the Select a VM size page, click the A-Series v2 dropdown and select A1_v2.

Then, click the Select button.

Step 25

From the Administrator account section, click into the Username textbox and type:

plab

In the Password textbox, type:

adminPassw0rd

Do this again for the Confirm password textbox.

Step 26

From the Create a virtual machine webpage, under Public inbound ports, check Allow selected ports.

Ensure the Select inbound ports dropdown box shows the RDP (3389) option as selected.

Under Licensing, tick the I confirm I have an eligible Windows 10 license with multi-tenant hosting checkbox.

Step 27

Scroll back up to the top of the Create a Virtual machine page.

Click Review & create at the foot of the page.

Step 28

Click the Review + create button.

Step 29

From the Validation page, click the Create button.

Step 30

The Overview page of the storage resource will load, and the Deployment is in progress information will be displayed.

Step 31

The Overview page will display Your deployment is complete when it’s finished.

Step 32

Click the Home weblink.

Step 33

Click the three-line icon at the top left.

Step 34

From the pop-out menu, select Dashboard.

Step 35

The My Dashboard page will appear. Click the All resources option.

Step 36

The All resources page will appear. Click the Add filter button on the right.

Step 37

From the Add filter pop-up box, select the Filter dropdown. under Tags, select Practice VM 01.

Step 38

Ensure that the Value is set to all, then, click Apply.

This will display only the resources associated with Practice VM 01 that we created earlier in our dashboard.

Step 39

Now from the All resources webpage in our dashboard, click on the Network Interface type that is associated with our VM.

In this example, it is listed as practicevm01892. This will show slightly differently for you.

Step 40

Now the Overview webpage will load for our practicevm01892 Network interface resource.

Click Metrics from the left-hand pane.

Step 41

Now the Metrics webpage will load for our practicevm01892 Network interface resource.

Click on the Metric dropdown box and select Bytes Received.

Step 42

You will now see the reported Bytes Received metrics displayed in the dashboard.
