Exercise 1 - Examining and Configuring Methods for Optimizing Cloud Environments

Cloud platforms offer us many options that allow us to custom-tailor our environments to meet the most demanding requirements and needs, but we must know what tools they provide to us and how to use them so we can design our environments accordingly.

57254ae0-eddf-432e-8472-186a49613f5b

In this exercise, you will learn about the concepts and methodologies we use to optimize cloud environments.

We will be going over the theories and tools we use for scaling, compute, storage, networking, and placement to create environments that are fault tolerant and highly available.

Learning Outcomes

After completing this module, you should be able to:

  • Explain right-sizing, compute, storage, network, and placement concepts in use for cloud optimization.

  • PLABWIN10 - Standalone Azure Workstation (Windows 10)

Task 1 - Configure Software Defined Networking (SDN)

In this task, we will create a form of a Software Defined Network (SDN) using Azure’s Virtual network creation tool. We will be able to configure the range of IP addresses we want to use and specify a subnet that can be used for VMs located in the UK South region.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, click Microsoft Edge.

Note: When opening the Microsoft Edge browser, it will automatically redirect to the Azure Portal and sign in. An Azure subscription will be provided for all lab activities.

Step 3

From the Azure portal page, click Virtual networks.

Step 4

From the Virtual networks page, click + Create.

Step 5

From the Create virtual network page, ensure the Resource group dropdown is showing the pre-populated resource group.

Step 6

From the Instance details section, click in the Name textbox and type:

practiceVN

Step 7

Click Next : IP Addresses >.

Step 8

From the IP Addesses tab, notice the IPv4 address space is already populated with 10.0.0.0/16.

Under IPv4 address space, click in the empty textbox and type:

10.1.1.0/24

Step 9

Click Next : Security >.

Step 10

From the Security tab, change the Firewall option to Enable.

Firewall fields will then appear. In the Firewall name textbox, type:

practiceFW

Step 11

In the Firewall subnet address space box textbox, type:

10.1.1.0/24

Step 12

Under the Public IP address dropdown, click the Create new weblink.

Step 13

From the Add a public IP address pop-out window, in the Name textbox, type:

publicpractice

Click OK.

Step 14

Notice the Public IP address dropdown is now populated with (New) publicpractice.

Step 15

Click the Review + create button.

Step 16

From the Review + create tab, click the Create button.

Step 17

The Overview page of the storage resource will load, and the Deployment is in progress information will be displayed.

Step 18

The Overview page will display the message Your deployment is complete when it’s finished.

Click Go to resource.

Step 19

You have now deployed a Virtual network; this is an SDN in its simplest form.
