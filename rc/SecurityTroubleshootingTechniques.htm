Exercise 1 - Examining and Performing Security Troubleshooting

In this exercise, you will go over security troubleshooting concepts associated with privilege, authentication, authorization, security groups, keys and certificates, misconfigured or misapplied policies, data security issues, exposed endpoints, misconfigured or failed security appliances, unsupported protocols, and external/internal attacks.

We will also troubleshoot users in a Ubuntu Server 22.04 LTS environment, granting certain users sudo access, changing passwords, and examining what users have sudo access.

Learning Outcomes

After completing this module, you should be able to:

Step 21

You will now see a prompt that says ubuntu@ubuntu00:~$.

Step 22

Now that we are logged into the Ubuntu server we want to see a list of all users on the system.

Type the compgen -u command to view a complete list of users:

compgen -u

Press Enter.

Notice the last four users, helpdesk01, helpdesk02, sysadmin01, sysadmin02. These are the users that we want to configure.

We will be granting the sysadmin users sudo privileges. We will make sure that the helpdesk users do not have sudo privileges.

Step 23

Now to find out exactly what users currently have sudo privileges type:

getent group sudo | cut -d: -f4

Press Enter.

Now we can see that the only user that has sudo privileges is the ubuntu user we are currently logged in as.

Step 24

Now you will add the sysadmin01 user to the sudo group, you will do this by typing:

sudo adduser sysadmin01 sudo

Press Enter.

If prompted to enter a password, use the following:

Passw0rd

Press Enter.

You will see the sysadmin01 user has been added to the sudo group.

Step 25

Now lets add sysadmin02 to the sudo group, type:

sudo adduser sysadmin02 sudo

Enter.

Step 26

Now lets clear the terminal window so that we don’t have so much information in front of us, type:

clear

Press Enter.

Step 27

Now to confirm what user we are logged in as use the whoami command, type:

whoami

Press Enter.

Now we can see that we are logged in with the ubuntu user, notice that it also shows in the prompt ubuntu@ubuntu00:~$ this is important to know as you might come across a linux server terminal that does not display the username in the prompt.

Step 28

Now lets switch users to the sysadmin01 user, type:

su - sysadmin01

Now we will need to provide the password for this user, type:

Passw0rd

Press Enter.

Step 29

Now to confirm what user we are logged in as use the whoami command, type:

whoami

Step 39

Now let’s see if we are able to make changes to a sudo user with the helpdesk01 user who does not have sudo privileges. We will try and change the password for the sysadmin02 user. To do this type:

sudo passwd sysadmin02

Press Enter.

Now you must provide the password of the current user you are logged in as, to do this type:

helpdesk

Press Enter.

Step 40

Notice that the response back from the terminal is helpdesk01 is not in the sudoers file. This incident will be reported. This is because the helpdesk01 user does not have sudo privileges.

Step 41

You can check what users do have sudo privileges by typing:

getent group sudo | cut -d: -f4

Press Enter.

Step 42

As you can see neither helpdesk01 or helpdesk02 users are listed as sudo users. Let’s switch users so we can perform sudo commands. We will now login as the sysadmin02 user, type:

su - sysadmin02

Press Enter.

Now we will need to provide the password for this user, type:

AdminPassw0rd

Press Enter.

Step 43

Press Enter.

Press the Enter button for each of these as this for informational purposes only and do not have any effect on our configuration.

Step 46

Now we will see that the terminal asks Is the information correct? [Y/n] type:

y

Press Enter.

Step 47

Now you will add the sysadmin01 user to the sudo group, you will do this by typing:

sudo adduser sysadmin01 sudo

Press Enter.

You will see the sysadmin01 user has been added to the sudo group.

Step 48

Now to confirm our new user has sudo privileges type:

getent group sudo | cut -d: -f4

Press Enter.

Step 49

You now see that sysadmin03 has been added to the sudo group.
