Task 1 - Automate Shutting Down Remote Systems with a Script

In this task, you will create a PowerShell script that will shut down the PLABDM01 device within the environment.

Step 1

Ensure PLABDM01 and PLABWIN10 are both powered on.

Connect to PLABWIN10.

Step 2

Click into the Type here to search textbox and type:

notepad

From the Best match results, click the Notepad app.

Step 3

Using Notepad, you will create a PowerShell script that will shut down the plabdm01 system within our environment.

In the Untitled - Notepad document, type:

Stop-Computer -ComputerName plabdm01 -Force

In our script:

Stop-Computer is the command used to shut down the system.

-ComputerName defines the system to be shut down, in this case, plabdm01.

-Force is a switch used in this script that tells the system, “we are sure we want to do this”.

Step 4

Click on File and select Save As…

Step 5

Click Desktop.

Click in the File name textbox and replace *txt with:

Stopcomputer.ps1

Click Save.

Note: Adding the .ps1 to the filename now makes this a PowerShell script that we can run.

Step 6

Once saved, close Notepad.

Step 7

Notice that the Stopcomputer.ps1 script icon on the Desktop.

Step 8

Click into the Type here to search textbox and type:

cmd

From the Best match results, click the Command Prompt app.

Step 9

From the Command Prompt window and type:

ping plabdm01 -t

Press Enter.

Note: This command sends a constant ping to the plabdm01 device. We will use this constant ping as an indicator of when plabdm01 will go down once we run our stopcomputer.ps1 script in the next steps.

Step 10

Confirm that you are receiving replies from plabdm01 192.168.0.2.

Keep the Administrator: Command Prompt window up while the pings are still running.

Minimize the Administrator: Command Prompt window.

Step 11

From the Desktop, right-click Stopcomputer.ps1 and select Run with PowerShell.

Step 12

Reopen the Command Prompt window.

Wait a few minutes and observe the change in response you will see in the Command Prompt.

Step 13

After a few minutes, notice the pings not responding. This is indicated by the Destination host unreachable response.

87fab35c-2831-4202-b4f1-46339fa6c815

Step 14

Close the Command Prompt window.
