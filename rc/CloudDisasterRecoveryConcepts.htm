Task 2 - Configure Network Settings

In this task you will configure the network settings of the. You will then test to make sure you are able to pass traffic to and from the VM.

Step 1

Ensure you are connected to PLABDM01.

Once you see a prompt that says ubuntu@ubuntu00:~$, type the following:

cd /etc/netplan

Press Enter.

Step 2

Type the following:

ls

Press Enter.

Step 3

The prompt will return the name of a file, this name will be used in this steps command.

In our example here, it returned 00-installer-config.yaml.

Use the name returned for you for the last part of the command.

Replace the xx-xx-xx with your file name details and type:

sudo nano xx-xx-xx

So it should look like this sudo nano 00-installer-config.yaml

Press Enter.

Note: If you are asked for a password, use the password you setup, Passw0rd

Step 4

Now you will see the contents of the file.

Step 5

Using the up and down arrows of your keyboard navigate to the dhcp4: field and delete true and, make sure you leave a space after dhcp4. Type:

false

Press Enter.

Then press the Tab key until the cursor is below and aligned with the d of dhcp4. Type:

addresses: [192.168.0.10/24]

Press Enter.

Then press the Tab key until the cursor is below and aligned with the a of addresses:. Type:

gateway4: 192.168.0.250

Press Enter.

Then press the Tab key until the cursor is below and aligned with the g of gateway4:. Type:

nameservers:

Press Enter.

Then press the Tab key until the cursor is below and aligned with the m of nameservers:. Type:

addresses: [8.8.8.8,4.2.2.2]

Press Enter.

Step 6

On your keyboard press key-combination of CTRL+X, you will be asked Save modified buffer?

Press the Y button on your keyboard.

Step 7

You will see File Name to Write: 00-installer-config.yaml.

Press Enter.

Step 8

You will now be back at the ubuntu@ubuntu00:/etc/netplan$ prompt.

From the ubuntu@ubuntu00:/etc/netplan$ prompt, type:

sudo netplan apply

Press Enter.

Step 9

You will be back at ubuntu@ubuntu00:/etc/netplan$ prompt and see the changes have been made. Type:

ip a

Press Enter.

Step 10

You will see the IP address of 192.168.0.10/24 listed for eth0, this confirms our configuration changes were successful.

Step 11

Now close out of the connection to ubuntu00. Click on the X button on the right-side of the blue bar.

Step 12

Right-click the Start icon and type:

command prompt

From the Best match results, select Command Prompt.

Step 13

From the Administrator: Command Prompt window, type:

ssh ubuntu@192.168.0.10

Press Enter.

Step 14

When asked to continue, type:

yes

Press Enter.

Step 15

When asked for the password, type:

Passw0rd

Press Enter.

Step 16

You will now see the command prompt change to ubuntu@ubuntu00:~$.

This means you have successfully logged into the ubuntu00 server.

Step 17

Type:

ping 8.8.8.8

Press Enter.

Note: Once you have confirmed you are getting replies you can now close the command prompt window by clicking on the X at the top right of the window.
