Exercise 1 - Cloud Shared Responsibility Service Models

Different cloud resources and infrastructures require different levels of responsibility for both the user and the Cloud Service Provider (CSP). This is known as the Shared Responsibility Model and allows users to give more responsibility to the provider depending on what sort of environment they are looking to build and vice-versa.

For example, a software developer may not be interested in a private on-premises infrastructure that requires networking knowledge to build and configure a networking environment and instead just be interested in a safe sandbox environment to test code deployments.

In this case, a Platform as a Service (PaaS) environment would suit their needs, and they can leave the networking and infrastructure responsibility to the CSP.

It's important to remember that with shared responsibility comes different subscription charges.

Note: Billing and subscriptions will be covered in a future module. f55c4819-8c68-4c15-8394-7286d43106f0

External to this is the On-Premises model, wherein full responsibility lies with the organization and all private cloud infrastructure.

However, It’s good to consider On-Premises and Software as a Service (SaaS) models as two sides of the same coin - the first gives full responsibility to the user, and the second offers a full service to users - including tools such as Microsoft 365, Twilio and even Netflix.

Exercise 2 - Advanced Cloud Services

Further to different service models, CSPs also offer different types of more advanced services.

These emerging technologies aim to further assist users with multiple things, from data aggregation and collection to fully managed Serverless architectures, and even Artificial Intelligence and Machine Learning tools to assist with the collation and processing of data for insights.

Picture you work for a home assistant provider (such as Amazon) and need to aggregate data from millions of different IoT devices across the globe, all reporting information and telemetry back to your Cloud infrastructure. Because of the sheer amount and volume of this data, it would be highly beneficial to implement Artificial Intelligence to help turn that data into usable metrics rather than manually process and sift through it.

AI is used for this and many more applications in Cloud environments and aims to provide users with insights and easily track user trends. AI can also process images and speech patterns to provide more and more insights, which cascades into more accurate reporting done by the intelligence.

There is a differentiation between Machine Learning and AI here; AI can analyze and produce insights, whereas Machine Learning gives users the ability to make decisions based on those same insights.

In this exercise, we will investigate some advanced and emerging technologies available in the Cloud.

Task 1 - Explore Cloud-based IoT Resources

The term IoT stands for the Internet of Things. IoT allows a device to be accessible on the Internet, such as a mobile device. IoT devices can communicate with each other and gather information that can be used for analysis. There are hundreds of different categories of devices covered under the IoT umbrella, such as wearable devices (smart watches), cameras, climate control, connected and smart cars, and home assistants such as Amazon’s Alexa.

Most modern IoT devices use Cloud technologies to communicate and relay data to users. A home assistant, for example, feeds data back on user trends to a cloud location for the provider, which in turn is then used to further develop and improve the effectiveness of the assistant on a per-user basis. Cloud technologies are better to implement here due to scalability - the sheer amount of data that a home assistant provider would have to process through all devices simultaneously is massive, which could be an issue if storing that data was done on physical hardware.

In this task, we will investigate IoT Hub, a tool available in Azure that allows management and communication to any connected IoT devices.

Task 2 - Explore Serverless Architectures

Serverless does not refer to infrastructures without any actual physical servers (data always needs to be stored somewhere), but instead an infrastructure that aims to take the management of general server maintenance and configuration out of the user's hands.

Picture a scenario in which a single software developer aims to build an application. If they implemented a Serverless cloud infrastructure to work from, they would not need to manually configure and build servers to host their code but instead have a scalable infrastructure that already runs in the background, freeing up the time and resources it would cost to build this on-premises. It can also cut down on overhead staffing costs - the developer in our example may not be a networking specialist, which may require someone with these skills to also be brought on board to the project.

Serverless architectures provide a fully managed solution that is on-demand (or pay-as-you-use). Examples include AWS Lambda, Azure Functions, and Google Cloud Functions. In this task, we will explore a Serverless architecture in a public cloud.
