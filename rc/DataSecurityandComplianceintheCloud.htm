Task 1 - Use HashCalc to Encrypt Files

Hashes are "one-way cryptographic" algorithms that mask data with various characters based on the output of that data processed through said algorithm. The two main hashes are MD5 and SHA.

HashCalc is a tool that allows you to check thirteen popular hash and checksum algorithms. In this task, you will learn to use HashCalc.

In this task, you will be using the HashCalc tool.

Connect to PLABDM01.

Step 2

hash

From the Best match results, select the HashCalc Desktop app.

Step 3

Go back into the Type here to search field, type the following:

notepad

From the Best match menu, click Notepad.

Step 4

Type the following in Notepad:

We are testing HashCalc!

Step 5

Click File > Save As.

Under Quick access, select Desktop.

In the File name textbox, replace the existing entry with:

Click Save.

Notice the hashme.txt icon on the Desktop.

Exercise 2 - Investigating Cloud Compliance Controls

It is key that anything created and stored inside of a Cloud is compliant with relevant regulatory standards. Most organizations have retention policies for data (such as Personally Identifiable Information or PII) to ensure that they are compliant. In the context of Cloud computing, data can be given a Retention Hold which means that any resource tagged this way will be retained for a time as specified by the hold, and then once expired, can be deleted. Typically, a management policy would be enabled that automatically deletes and destroys data that has passed a retention period.

Data can also be tagged by Legal Holds, which ensures that data is not deleted or overwritten. In public cloud subscriptions, you can add this tag to many different types of resources, such as an S3 Object Lock in AWS (which has options for either Legal or Retention Holds). This ensures that the locked resource cannot be deleted or overwritten as long as the lock is applied. Another term for this is a Write Once Read Many (WORM) drive, which is a storage device that only allows data to be written onto it and retrieved for reading, but excludes anyone from editing that data, useful for archiving purposes.

These different options are part of what is known as Records Management, a term that refers to managing data for business critical and legal purposes. These policies exist to enforce compliance with regulations, as well as ensure that the most recent and up-to-date versions of documentation are available at any time.

Part of this is ensuring that data kept for regulatory purposes (typically documentation such as legal charters, business proposals etc.) are correctly versioned to ensure that all data modifications can be tracked, and that the most up-to-date version of that data is available for anyone that might need it.

Legal documentation can also be provided directly by CSPs for public clouds in the instance that a regulatory body or auditor needs to verify a requirement is being met. AWS Artifact, a database which stores Amazon’s internal, can be accessed by users with specific privileges, which can, in turn, be shared to ensure compliance.

Data Loss Prevention (DLP) is a policy that exists to protect sensitive data, which is generally any information that can be used to track down to a specific user. This data should be protected against any unauthorized sharing, such as third-party attacks. DLP should be implemented at the policy level (such as content analysis policies) alongside data security controls, to ensure that sensitive information is tagged as such and is protected.

A Cloud Access Security Broker (CASB) is a service available in public cloud subscriptions that act as an agent to streamline all of these processes together. A broker will handle all information, streamline cloud access security, provides visibility of secure applications. An example of this is SkyHigh Security CASB in AWS.

In this exercise, we will investigate AWS Artifact, S3 Object Locks, WORM storage and how to implement a Cloud Access Security Broker (CASB) in AWS.

Task 1 - Investigating AWS Artifact

AWS Artifact is a tool that allows users to download Amazon’s security and compliance documentation to be shared with auditors and regulators. This allows users to demonstrate compliance with resources held inside of their AWS public cloud subscription.

In this task, we will investigate how to navigate to and use AWS Artifact.
