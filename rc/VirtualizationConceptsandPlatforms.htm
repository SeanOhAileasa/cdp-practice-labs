Welcome to the Virtualization Concepts and Platforms Practice Lab. In this module, you will be provided with the instructions and devices needed to develop your hands-on skills.

Virtual Machines (VMs) are created using Hypervisor software, and when working in private cloud infrastructures there are multiple considerations to consider when looking to provision VMs. Hypervisors can be one of two types, known as Type 1 and Type 2 hypervisors.

Type 1 refers to a hypervisor that runs on a host’s physical hardware, otherwise known as a “bare metal” configuration.

In this type of virtualization, the Hypervisor does not rely on the operating system used by the host machine and instead makes calls directly to hardware to generate virtual resources, and an example of this would be VMWare ESXi.

This is the opposite with Type 2 hypervisors, which run “on top” of an operating system and rely on that OS to make calls to the system hardware in its place. An example of this type of configuration would be Virtual Box.

In this module, a Type 1 hypervisor, Microsoft Hyper-V, will be used to explain virtualization concepts.

Task 3 - Provisioning a Virtual machine for Oversubscription

The Hyper-V application in Windows 10 can be used with virtualized machine resources to demonstrate Oversubscription of shared resources. This oversubscription of resources happens when the Hypervisor shares with virtual machines beyond the available physical resources on the host hardware.

Oversubscription of CPU cores isn’t a new concept; this capability is largely attributed to Simultaneous Multi-Threading (SMT). SMT is the technique used in modern processor designs that enable the processor's resources to be shared by separate, independent execution threads. Generally used to run parallelizing computations to increase instruction throughput, this also allows for several virtual machines to run on a single host.

Among other resources that can be shared are memory and local storage. Normally when configuring these resources for a virtual machine, they would be statically set; however, there are instances where it's necessary to allocate it dynamically, such as having hardware limitations or optimizing currently available resources. Hypervisors can manage this type of resource sharing using techniques such as Ballooning and Dynamic Memory Allocation.

In this task, the Hyper-V application will be further explored to get a better understanding of some common virtualization concepts.

Note: Simultaneous multi-threading (SMT) is what allows for the hosting of multiple virtual machines on a single host. Having provisioned these two virtual machines on a single host represents a simple form of hyper-converged infrastructure. This solution combines storage, computing, and networking into a single system reducing complexity and increasing scalability. Without the ability to virtualize this infrastructure, it would have been necessary to obtain, configure and connect multiple physical assets.

This speed of 2.60GHz is the computer’s processor clock speed, which determines how quickly the CPU can retrieve and interpret instructions.
By setting both Virtual machines to use the 4 cores provided, the CPU is considered Oversubscribed, as the CPU has to support these two VMs. In essence, it allows the host machine to retain both VMs without expanding the physical CPU, which would add costs but will require the administrator to assess for degraded performance beyond acceptable levels. Note: Glancing at the setting available on the Memory page, Under Dynamic Memory with it enabled, it allows for a Minimum and Maximum utilization of Ram. Despite the fact that 4096 MB has been statically set to this Virtual Machine, enabling Dynamic Memory allows for the Hypervisor to perform memory ballooning.

This allows the Hypervisor to expand the memory pool artificially by using or reclaiming unused memory assigned to the hosted virtual machines. Setting the memory to weigh in this instance notifies the Hypervisor to prioritize this VM when allocating memory. Depending on the utilization across all the virtual machines, it is possible to exhaust available memory if all the VMs simultaneously require this memory, reducing the overall performance of the systems.

This is possible through GPU Passthrough, where the device acts as if it was directly physically attached to the VM. Commonly used to run a Windows virtual machine that requires a dedicated GPU for an application to operate at native speeds.
