Exercise 1 - Exploring Cloud Deployment Models

In its simplest terms, the cloud is a vast network of devices used to pool resources together for use by end-users for many different applications, such as general file storage to hosting thousands of servers.

Due to the many attractions of cloud environments, many companies exist that provide these services, known as Cloud Service Providers (CSPs). The most popular CSPs are Amazon Web Services (AWS) and Microsoft Azure, both owned and operated by two US-based corporate monoliths, Amazon and Microsoft.

Furthermore, there are three different types of cloud, known as deployment models.

12938fdd-aa8d-4f2c-ac16-cd90afbc2a5d

An option when building an infrastructure is nesting a Virtual Private Cloud (VPC) inside an existing cloud, which then performs the same function as a separate cloud solution. This is known as Cloud within a cloud, and is very popular in real-world Private infrastructures due to the lower overhead costs. For example, it is common for businesses to host a private cloud using resources provided by a public cloud subscription. These types of cloud are not used by multiple tenants.

Similarly, there are also options for organizations to use multiple types of cloud at once, known as Multicloud, typical with Community clouds. These solutions typically combine different services and deployment models depending on what is needed at any given time. For example, an organization such as a school could host Software as a Service (SaaS) resources as part of an organization-wide cloud subscription used by the faculty, and then utilize a separate cloud for students.

Note: Different service models (such as Software as a Service) will be discussed in a future module.

There is also the concept of Multitenancy, in which different users of a private cloud provider access the same computing resources. This is synonymous with Public solutions, in which different customers around the world are utilizing the same servers to create their resources, although each user (or tenant)â€™s data is kept fully separated and isolated from others using the same physical hardware.

In this exercise, we will investigate an example of a public cloud in Microsoft Azure.

Task 1 - Navigating the Microsoft Azure Portal

The Azure portal is a web interface used to create and manage Azure cloud resources and is an example of a Public cloud infrastructure.

In this task, the Microsoft Azure portal will be explored to better understand how to create and maintain cloud infrastructure resources.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, select Microsoft Edge.

Note: When opening the Microsoft Edge browser, it will redirect you to the Azure portal and sign in automatically.
An Azure subscription will be provided for all lab activities that utilize the Azure platform.
Note that in a real-world scenario, either a subscription will need to be purchased or will already exist as part of existing company infrastructure.

Step 3

In the top left-hand corner of the Azure portal, select Show portal menu

Step 4

From the Portal menu, select Dashboard.

Note: The Dashboard page gives an overview of all resources and services that have been created for the subscription. The Dashboard can be customized according to a user's needs by adding shortcuts to resources that might be used more often than others.
As this subscription contains no resources, the Dashboard will be empty. In a real-world environment, you will see multiple resources for multiple resource groups, depending on your role and internal permissions.

Step 5

From the My Dashboard page, click the show portal menu again and select All services.

Step 6

In the left-hand pane of the All services page, under the Categories section, select General.

Note: The General heading in Azure will list popular tools, such as Resource Groups, Marketplace and Service Health.

Step 7

In the same left-hand pane, select Compute.

Note: In the Compute section, various options are available for creating computing resources in the Azure infrastructure.
These resources are classified as Infrastructure as a Service (IaaS) which will be investigated more in a future module.

Step 8

In the left-hand pane, select Networking.

Note: In the Networking section, different virtual networking components are listed. These components connect virtual devices to facilitate communication between these devices.

Step 9

In the left-hand pane, select Storage.

Note: In the Storage section, various components storage components of the Azure platform are displayed. These storage components include creating storage for Virtual machines or creating storage that will be accessible in the cloud.

Step 10

In the left-hand pane, select Databases.

Note: In the Databases section, various options are available to create an infrastructure database. These can be classified as Platform as a Service (PaaS). When databases are created, there is no need for a server infrastructure as this will be maintained by Microsoft.

Step 11

In the left-hand pane, select Web.

Note: In the Web section, the available web application services that can be created on the Azure portal is shown. A company would create a web application hosted on Azure instead of acquiring and maintaining the physical infrastructure to host web servers. Microsoft manages and maintains these servers in the backend without any customer intervention.

Exercise 2 - Exploring Private Clouds

Outside of utilizing a public cloud subscription, an organization could also investigate ways to develop a private cloud infrastructure depending on their needs as a business.

For example, a company that provides a web service or application to its user base may not wish to relinquish all trust to a third-party, especially around security and uptime areas. Therefore, it may pursue a private infrastructure instead. It entirely depends on the needs of the business.

Instead of purchasing a subscription, an organization could instead invest in physical hardware (typically multiple servers inside of a datacenter) to use as a platform for creating resources used by their employees or end-user base.

In this exercise, we will investigate and create a simple private cloud infrastructure using Hyper-V on Windows Server 2019.

Task 1 - Configuring a Private Cloud Infrastructure in Hyper-V

Using tools readily available on Windows Server 2019 (a typical operating system used in Datacenters), a private cloud can be created using virtualization software, such as Hyper-V. These types of machines are known as Virtual Machines (VMs).

In this task, an example of how to create a simple private cloud infrastructure will be explored.

Step 1

Power on PLABDM01.

The Server Manager dashboard will open by default.

Note: As discussed above, Windows Server is a very popular operating system used in Datacenters as a foundation for private cloud infrastructures.

Step 2

On the right-hand side of the page, navigate to the Tools drop-down menu and select Hyper-V Manager.

Step 3

The Hyper-V Manager window will now be displayed.

Note: Virtualization software such as Hyper-V allows users to easily create and spin up virtual machines, which can be used in multiple different ways.
Picture this, instead of connecting to a server inside of a typical office environment, you have instead remotely connected to a server inside of a datacenter that different people inside of your organization can access.
Therefore, any device created here can be used for multiple applications, making this a private cloud infrastructure.
