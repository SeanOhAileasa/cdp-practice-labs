Exercise 1 - Examining and Configuring a Monitoring Solution

Network monitoring is the methodology in which you monitor cloud and infrastructure resources and measure their performance. This monitoring provides insight into uptime, network traffic, CPU usage, memory utilization, storage consumption, and other key data.

e60e1b64-6ce4-4324-9e20-846b30129118

In this exercise, you will learn about the foundations of monitoring, then install and configure a monitoring solution. Then finally, you will examine metrics using the monitoring solution.

  • Define and Describe metrics in a Monitoring Environment

Task 1 - Set up a Monitoring Solution

In this task, you will open and set up the OpManager application to create a monitoring solution for a private cloud infrastructure.

Step 1

Connect to PLABWIN10.

Step 2

From the Taskbar, click the File Explorer icon.

Step 3

Navigate to This PC > Local Disk (C:) > Software folder.

Double-click the app ManageEngine_OpManager_64bit.

Step 4

The installation page will appear.

Step 5

You will see the Windows Security Alert box, and untick Private and public networks checkboxes.

From the Windows Security Alert dialog box, click Allow access.

Step 6

From the ManageEngine OpManager installer screen, click on the Next > button.

Step 7

When you see the OpManager License Agreement screen. click on the Yes button.

Step 8

From the Destination Folder screen, click on the Next > button.

Step 9

From the Port Selection Panel screen, keep the WebServer as 8060.

Click on the Next > button.

Step 10

You will see the OpManager Registration screen. Click on the Skip button.

Step 11

You will see the OpManager Installation Progress screen; wait for this to complete.

Step 12

You will see the OpManager Select backend database screen, select POSTGRESQL and click on the Next > button.

Step 13

You will see the OpManager InstallShield Wizard Complete screen. Click on the Finish button

Note: If the OpManager Antivirus scanner warning screen appears, click OK.

Step 14

You will see the OpManager Initializing screen; wait for this to finish.

Step 15

The OpManager log in page will appear.

Keep the OpManager window open to refer back to later in this task.

Task 2 - Use Monitoring Solution

In this task, you will set up a simple network monitor on a Windows 10 machine that will provide you with data about a Windows Server 2019 Domain Controller.

Step 1

Connect to PLABDC01.

Step 2

Once you are connected to PLABDC01, we need to find out key information so we can cross-reference this against what we will see in OpManager.

Click the magnifying glass icon in the Taskbar and type:

cmd

Select Command Prompt from the Best match results.

Step 3

Type the following:

ipconfig

Press Enter.

Note: The results are displayed showing the vEthernet adapter IPv4 address of 192.168.0.1

Step 4

Type the following:

hostname

Press Enter.

Step 5

Finally, type in the following:

whoami

Press Enter.

Step 6

We now have the IP address, hostname, and the user name that you are logged in with. With this information, we can now reconnect to PLABWIN10.

Step 7

Reconnect to PLABWIN10.

Type in the following login details:

Username

Admin

Password

Admin
Note: As this is a testing environment, you don't have to change the password, but keep in mind that in a production environment, you will want to secure this login with your own password.

Click the Login button.

Step 8

Once you are logged in, click X to close the Get started in 5 simple steps pop-up window.

Step 9

In the address bar at the top of the browser, you can see the hostname of this machine with colon 8060. That is the port number.

Step 10

From the top menu, click Inventory.

Step 11

Notice that our three systems are populated.

If all three haven't populated yet, refresh the page until they appear. This may take a few attempts.

Step 12

Now, from the listed systems, under the column Device Name, click Plabdc01.

Step 13

The Plabdc01 page should appear. You will see an option to add credentials.

Next to the Credentials listing, click the Click here to change weblink so we can add the credentials that were provided to you so OpManager can start pulling data.

Step 14

The Device Credentials window appears. Drag the slider to view the whole window if needed.

You will need to input the user, including the domain name. We get this information from the whoami command that we ran earlier. Type the following:

Domain Name \ User Name

practicelabs\administrator

You will need to input the password that has been provided to you. Type the following:

Password

Passw0rd

Click the Test Credential button.

Step 15

Now you can see that the credentials are working and notice that WMI passed.

However, SNMP says Not Tested. That's because we have added Windows credentials, so we are pulling WMI information in this lab.

If we had a network device such as a switch or firewall, we could enable SNMP on those devices and pull data, but for this exercise, we will stick to WMI.

Step 16

Now that we have successfully added our credentials, scroll down the page and click OK.

Step 17

Reconnect to the PLABDC01 device.

Step 18

Reconnect back to PLABWIN10.

Click on Monitors tab.

Step 19

Notice that data is being pulled. You should see something like this:

Step 20

Now, scroll to the right and click the edit button on CPU Utilization.

Step 21

Change the Interval (Minutes) to:

1 minute

Step 22

Scroll down and select Save.

Step 23

Click on Memory Utilization.

Step 24

Once you have done that, click on the name of the monitor and actually see some data.

Step 25

Click back on Inventory.

Step 26

Select PLABDC01.

Step 27

Ensure you are on the Summary tab.

Notice the Availability is at 100%.

This is because as long as OpManager has been monitoring PLABDC01, it has not shut off. If we were to turn off PLABDC01, we will see this turn red.
