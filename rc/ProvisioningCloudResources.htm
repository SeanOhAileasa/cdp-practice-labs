Exercise 1 - Provisioning Storage Resources

With any given environment, retaining enough storage to manage an organization’s data needs, took delicate planning. Leveraging Azure’s cloud storage platform provides system administrators more flexibility in adjusting to the ever-fluctuating data requirements presented by their organizations.

Additionally, Azure storage offers various options for data redundancy and availability, without the additional strain of planning out physical hardware requirements like an on-perm environment.

In this exercise, you will provision a storage account and create a blob storage.

Learning Outcomes

After completing this exercise, you should be able to:

  • Create a Storage Account in Azure
  • Create a Container and Set Access Permissions
  • Upload a File to Blob Storage and Review Metadata

Task 2 - Create a Blob Container

In the previous task, you created a storage account; this task will explore further into the provisioning of a blob container. This is Microsoft’s object storage solution, optimized for storing massive amounts of data.

Objects in Blob storage, with the right permissions configuration, can be accessed anywhere using HTTP or HTTPS.

Additionally, system administrators can even use Azure Storage Explorer to connect to the storage resource and manage it like it was on a local machine.

In this task, you will provision a blob storage.

Exercise 2 - Integrating Template Solutions and Containers into a Cloud Solution

Azure allows for a very modular approach to creating and configuring your environment, as various resources can be created on Azure cloud as individual instances. The utilization of the native templates, whether for OS image or individual solutions, allows for quick and consistent deployments.

This approach allows for individual instances independent of fully constructed Virtual Machines. Implementing only the resources needed for any given solution. Traditionally, virtual machines would be deployed with all resources needed for a given functions as well as resources that might not be used at a given point in time.

In this exercise, you will provision an app service and configure autoscale for this service.

Task 2 - Configure an Autoscale App Service

A useful feature available to Azure App Services is the ability to scale out the required resources to handle fluctuations in environment needs.

Implementation of this feature doesn’t require a change in the code the App service would be hosting, or even to redeploy the application itself.

This task will cover the utilization of Scaling out, which increases the number of VM instances that run the hosted application.

Step 1

Ensure you are connected to PLABWIN10 with the Azure portal window open.

Navigate to the app service cloudplusapp by clicking the cloudplusapp | Networking breadcrumb weblink.

Step 2

Under Settings, select Scale out (App Service plan).

Step 3

From the Configure tab, select Custom autoscale.

Step 4

Scroll down to the Auto created scale condition section, and ensure that the Scale based on metric checkbox is selected.

From the Instance limits > Maximum textbox type:

2

Then, from Rules, click on the Add a rule weblink.

Step 5

From the Scale rule flyout window, leave all the selected defaults and click Add.

Note: Reviewing over the configuration listed, when the CPU usage on the application sustains over 70% usage for 10 minutes or more, an additional VM will be deployed to handle the additional workload.
