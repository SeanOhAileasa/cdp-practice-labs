Exercise 1 - Examining the Troubleshooting Model

When an IT issue arises, the CompTIA troubleshooting methodology can be used as a guide for the best course of action. The steps are:

  • Identify the Problem
  • Establish a Theory
  • Test the Theory
  • Establish a Plan of Action

Notice that no changes have been made as yet. In a real-life scenario you would need to follow your change management procedures and policies in your organisation before you proceed.

Then, you would continue to:

  • Verify Full Functionality
  • Document

It can be difficult when working out an issue. The users with the issue may have written down the problem on their end, but the problem can be totally different from for information provided.

If a user comes up to you and informs you that “the internet is down”, this could be anumber of different issues and you must investigate and gather more information in order to pinpoint what the actual issue is at hand.

The user might think the internet is down, however, the real problem could be something as simple as their ethernet cable came unplugged.

The same can be said for an issue a user reports, it may turn out to be a much larger issue such as they can’t print to a network connected printer and the issue turns out to be a network switch that has locked up and all users connected to it cannot pass traffic.

In this exercise, you will run down the process of troubleshooting a common cloud-related issue from beginning to end. We will use a hypothetical situation and screenshots that you will likely see in real-life scenario.

Task 1 - Resolve a Policy Conflict in Azure

Before you proceed with troubleshooting a cloud-related issue, you will need to set up to lab environment.

In the following scenario, you have been asked to create a storage account by following the organisations’ policy.

As with any change, you can also get human error. So, you will not applying the region specified in the request and use the storage account in the following task.

In this task, you will set up a storage account within Microsoft Azure and resolve a policy conflict.

Scenario

Your Manager has sent a request for a new storage account resource in the company’s Microsoft Azure Portal.

Following the policy, the storage account should have the following applied:

  • Name = must be in lowercase using letters and numbers
  • Region = storage accounts should be located in the (Europe) UK South region
  • Performance = Standard performance
  • Redundancy = Locally-redundant storage
  • Advanced = Enable blob public access and key access.
  • Network access = Enable public access from all networks
  • Recovery = Disable point-in-time restore for containers and disable soft deletion.
  • Encryption and Tags = default state

Task 2 -Troubleshoot a Storage Account Issue

When reviewing an issue, start with identifying the problem.

Question the user changes - check what the user is experiencing and see if they can demonstrate the issue. Also verify if any changes have happened recently.

Investigate scope - identify who is affected or what is it affecting so you can start to isolate the scope.

Isolate the problem service - isolate the overall problem service as much as possible. You won’t be able to troubleshoot the whole cloud infrastructure, but once the problem service is isolated, you can start to refine the scope of what the nature of the issue might be.

Next, you will need to establish a theory of probable cause.

How do you believe it occurred - learn how it happened and got to this point.

Come up with every possible scenario. don’t assume you know the answer before looking at all the possible reasons for the fault. Create a list for you to review first, then prioritize your theories.

Pare down to most probable scenario by refining your list to 5-10 possible scenarios.

Check if the symptoms fit the theory, then, test the theory to determine cause.

  • Validate or invalidate the theory
  • If validated, determine next-step toward a solution
  • If invalid, reset and get a new theory
  • Escalate

Once you have all the information you need, you can begin to establish a plan of action, starting with performing backups before any changes and considering policies, and procedures along with the impact your proposed change will make.

Ensure you verify full functionality. It’s worth checking that everything is working as expected.

Finally, document the issue and resolution to help your company build up a knowledge base to speed up resolving similar issues in the future. It should include:

  • Findings
  • Actions
  • Outcomes

In this task, you will walk through the steps taken to resolve an issue in Microsoft Azure.

Scenario

An IT Service ticket has been logged stating the storage account you created recently will not allow blob storage, and there seems to be an issue with access keys.

Step 1

Ensure you are connected to PLABWIN10, and the Azure portal is open.

Step 2

Under Resources, click plstorage100.

Step 3

From the plstorage100 page, under the Security + networking menu, select Access keys.

Step 4

Notice on the plstorage100 | Access keys page, that a yellow warning message is showing.

It states that the authorization with Shared Key is disabled for this account.

Step 5

Under the Data storage menu, click Containers.

Step 6

Click Container.

Step 7

The New container fly-out window will display.

Notice that the Public access level dropdown is greyed out and a message below it states that public access is disabled on this storage account.

Click Discard.

Step 8

From the plstorage100 | Containers page, click the Overview menu option.

Step 9

From the plstorage100 page, you will be able to review the blob and key configuration to see why they aren’t working as expected.

Notice that both Blob public access and Storage key access settings are shown as Disabled.

To resolve the issues raised, both settings need to change to Enabled.

Click the Security weblink.

Important: Before making any changes to your Azure environment in a real-life scenario, ensure you follow the change management policies and processes in place within your company.

Step 10

From the plstorage100 | Configuration page, under Allow Blob public access, select Enabled.

Under Allow storage account key access, select Enabled.

Click Save.

Step 11

Let’s verify the change has resolved the access key issue.

Under the Security + networking menu, select Access keys.

Step 12

Notice on the plstorage00 | Access keys page, that denied access message is no longer shown.

Under the Data storage menu, select Containers.

Step 13

Let’s verify the change has resolved the blob issue.

Click Create.

Step 14

From the New container fly-out window, in the Name textbox, type:

plblob100

Notice that the Public access level dropdown is active. Ensure it’s set to Private.

Click Create.

Step 15

Notice that the plblob100 container created successfully.
