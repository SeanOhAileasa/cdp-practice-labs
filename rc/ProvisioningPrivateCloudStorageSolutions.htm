Exercise 1 - Configuring RAID Storage on a Virtual Machine

When considering storage available in the cloud, there are different types of storage that can be utilized for different types of data, such as:

Block storage: This type of storage allows data to be distributed onto different drives or partitions, where each block will be an individual hard drive. Provisioning this type of storage provides redundancy and avoids a single point of failure. An additional use of block storage is the incorporation of RAID for further redundancy or an increase in performance. Block storage is commonly used with a Storage Area Network (SAN), where the access the data across the network from a centralized device. Zoning of the data can be used to classify the data according to the type or location.

<b>File storage</b>: This type of storage is accessed on the network and is normally created on a local device or through a file server rather than spreading it across different devices or hard drives. This can be typically used on Windows devices, and the file share will appear on the device as a hard drive. This storage environment is known as Network Attached Storage (NAS).

Storage redundancy comes in many forms, with RAID configuration being a common approach to protecting sensitive data.

RAID uses an error protection scheme known as “parity”, to provide fault tolerance in a set of data.

Task 1 - Configure RAID 0

Despite the fact that RAID 0 does not provide information redundancy, or fault tolerance, its general purpose is to increase performance or used to create a large logical volume.

This configuration is primarily used when high performance is required, and an application can tolerate lower reliability. RAID 0 can be configured using two disks, of varied sizes, but the array will be limited to the size of the smallest disk.

Pairing RAID 0 and RAID 1 grants the benefits of high performance and a layer of redundancy; the trade-off is that it requires a minimum of four disks to set up.

A drawback to keep in mind when utilizing RAID 0 is that without redundancy, if one drive fails, it can cause the entire array to fail. Having data striped across all disks can result in complete data loss from a single disk failure.

Task 2 - Configure RAID 1

Raid 1, 5, 6, and 1 0 are where failure redundancy is addressed. Based on the Raid configuration that is chosen to be set up, will determine the technique that is employed.

Raid 1 is known as mirroring, requiring a minimum of 2 disks to configure. Using this configuration allows data to be replicated across the selected disks. In the event of a hardware failure of one of the disks, the remaining disk still retains a full copy of the data.

Raid 5 is known as distributed parity, requiring a minimum of 3 disks. A parity is a set of computations used for fault tolerance by calculating the data that is stored in the first 2 drives and storing the results in the third. In the event of a disk failure, the array status will change to degraded, but the array will remain functional.

Raid 6 is known as dual parity, requiring a minimum of 4 disks. Raid 6 extends the allocation of its parity disk to 2 and can withstand a 2-disk failure. Despite the added benefit of enhanced redundancy, Raid 6 suffers from reduced write operations, as its associated parity calculations.

For added resiliency, Raid configurations can be nested. Raid 1 0 is a simple example of this as it combines both striping and mirroring. The only shortcoming of nesting these raid configurations is decreased capacity utilization.
